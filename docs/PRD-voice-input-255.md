# Octo 语音输入 PRD

> GitHub Issue: https://github.com/dmwork-org/dmwork-PM/issues/255

## 背景 & 目标

增加语音输入、语音编辑、按住说话三种交互，降低文字输入成本。

## 需求概述

| 模块 | 功能点 |
|------|--------|
| 移动端 | 1. 话筒点击输入 2. 话筒长按编辑 3. 语音模式（按住说话） |
| Web 端 | 1. fn+space 短按 / 点击话筒输入 2. fn+space 长按编辑 3. Esc 取消 |
| 共用 | 1. 悬浮状态气泡 2. 转写失败处理 |

## Web 端 - 语音输入

| 手势 | 面板状态 | 松手结果 |
|------|----------|----------|
| 短按 fn+space（< 400ms 松开）或点击话筒图标（Mac/Win） | 悬浮气泡显示「语音输入」，话筒激活 | 开始录音 |
| 再次按 fn+space 或再次点击话筒图标 | 气泡显示「转写中」 | 停止录音，气泡消失 |
| 支持多次操作累积，每次转写文字插入到光标处 | | |

## Web 端 - 语音编辑

### 模式一：全文编辑（无选中文本）

1. 长按 fn+space（≥ 400ms，不松开）→ 悬浮气泡显示「语音编辑」（紫色）+ 话筒激活 → 开始录音（Mac/Win 通用）
2. 松开 fn+space → 停止录音 → 气泡显示「编辑中」→ 将「原输入框全部内容 + 语音转写文字」拼接后调后端 LLM 接口（系统预设固定 prompt，不可用户配置）处理；output 文字替换输入框全部内容 → 气泡消失

### 模式二：选中文本编辑（有选中文本）

1. 用户先在输入框中选中部分文字
2. 长按 fn+space（≥ 400ms）→ 悬浮气泡显示「语音编辑」（紫色）+ 话筒激活 → 开始录音（Mac/Win 通用）
3. 松开 fn+space → 停止录音 → 气泡显示「编辑中」→ 将「选中文字 + 语音转写文字」拼接后调后端 LLM 接口处理；output 文字仅替换选中部分，输入框其余内容不变 → 气泡消失

| 触发条件 | 替换范围 |
|----------|----------|
| 长按 fn+space，无选中文本 | 替换输入框全部内容 |
| 长按 fn+space，有选中文本 | 仅替换选中文字 |

**Esc 取消：** 录音中（包括语音输入及语音编辑）按 Esc → 取消录音 → 气泡消失 → 话筒 icon 恢复正常 → 输入框内容不变（Mac/Win 通用）

## 移动端 - 按住说话

| 手势 | 面板状态 | 松手结果 |
|------|----------|----------|
| 按住 | 蓝色，提示「松手发送  上移取消」 | 正常松手 |
| 上滑 80px+ | 变红，提示「松手取消」 | 取消 |
| 下滑回 80px 以内 | 恢复蓝色 | 正常松手 |
| 正常松手 | 面板收起 | 发送语音条和文本消息 |

## 移动端 - 语音输入

| 手势 | 状态 | 结果 |
|------|------|------|
| 点击话筒 icon | 悬浮气泡显示「语音输入」，话筒激活 | 开始录音 |
| 再次点击话筒 | 气泡显示「转写中」 | 停止录音，气泡消失；调后端转写接口 |
| 支持多次操作累积，每次转写文字插入光标处 | | |

## 移动端 - 语音编辑

| 手势 | 面板状态 | 松手结果 |
|------|----------|----------|
| 长按话筒 icon（≥ 400ms） | 悬浮气泡显示「语音编辑」，话筒激活 | 开始录音 |
| 松手 | 气泡显示「编辑中」 | 停止录音，气泡消失；将「原输入框内容 + 语音转写文字」拼接后调后端 LLM 接口处理 |
| 长按过程中上滑 60px+ | 气泡文案切换为「松手取消」 | a. 保持上滑状态松手 → 取消编辑 → 输入框内容不变 |

## 录音时长限制

- 最长录音时长：300 秒
- 达到 300 秒时自动停止录音，进入转写流程（行为同正常松手）
- 接近上限时（剩余 5 秒）不做额外提示，直接到时停录

## 转写服务说明

- 语音转写调后端服务处理，不在客户端本地执行
- 无网络时：话筒 icon 不可点击（置灰），点击时 Toast「网络不可用，无法使用语音功能」
- 语音编辑同理依赖后端 LLM 接口，无网络时同样不可用

## 录音结果写入规则

| 模式 | 写入方式 |
|------|----------|
| 语音输入（移动端 / Web） | 文字插入光标当前位置 |
| 语音编辑（移动端 / Web） | 无选中文本：替换输入框全部内容；有选中文本（Web 端）：仅替换选中部分 |
| 按住说话（仅移动端） | 松手后直接发送语音条 + 转写文字 |

## 异常处理

1. 转写失败：Toast「转写失败，请重试」，恢复可操作状态，输入框内容不变
2. 录音时长不足 1 秒：Toast「未检测到语音」，不触发转写
3. 麦克风权限被拒绝：展示对应平台引导授权提示
4. 无网络：话筒 icon 置灰，点击时 Toast「网络不可用，无法使用语音功能」

## 暂不做

- 多语言语音识别切换（后续版本考虑）
- 语音编辑的系统 prompt 用户可配置
- 语音消息接收方气泡的详细展示布局调整

## 验收标准

1. 移动端点击话筒，气泡显示「语音输入」；再次点击，气泡显示「转写中」；完成后文字插入光标当前位置
2. 移动端长按话筒 400ms，气泡显示「语音编辑」（紫色）；松手后气泡显示「编辑中」；完成后文字替换输入框全部内容
3. 移动端长按话筒上滑 60px+ 松手，输入框内容不变，Toast「已取消编辑」
4. 移动端切换至语音模式后，按住说话弹出蓝色面板；上滑 80px+ 面板变红；正常松手后发送语音条 + 转写文字
5. Web 端短按 fn+space 或点击话筒，气泡显示「语音输入」；再次操作停止录音，气泡显示「转写中」；完成后文字插入光标处
6. Web 端长按 fn+space 400ms（无选中文本），气泡显示「语音编辑」（紫色）；松开后气泡显示「编辑中」；完成后文字替换输入框全部内容
6b. Web 端先选中部分文字，再长按 fn+space 400ms，触发语音编辑；完成后仅选中部分被替换，其余内容不变
7. Web 端录音中按 Esc，取消录音，气泡消失，输入框不变
8. 转写失败时 Toast「转写失败，请重试」，输入框不变
9. 录音时长 < 1s 时 Toast「未检测到语音」
10. 录音时长达到 300 秒时自动停录，进入转写流程
11. 无网络时话筒 icon 置灰，点击 Toast「网络不可用，无法使用语音功能」
12. 麦克风权限被拒绝时展示对应平台 Toast 文案
